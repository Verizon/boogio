'''
dispatch.py

Manage dispatch schema instances for Opticon reports.

The jsonschema schemas for this package require
http://pypi.python.org/pypi/rfc3987/ and
http://pypi.python.org/pypi/strict-rfc3339. Install with

    pip install rfc3987
    pip install strict-rfc3339

How to use the Dispatch class:

The Dispatch class ensures that the proper fields are included in
elements of the dispatch list. The Dispatch schema
is a dictionary with three top level keys:

  * _meta -- specification of the json schema.
  * origin -- information regarding the creation of the dispatch, such
    as timestamps, servers involved, and scripts or tools used.
  * commentary -- descriptive copy provided for use in reports and
    other "downstream" presentations of the dispatch.

Dispatch instances store the dispatch record in the internal _dispatch
variable. The value in this variable will be checked for schmema
compliance in the following circumstances:

  * When the instance's is_valid() method is called.
  * When the instance's validate() method is called.
  * When the instance's json_dumps() method is called without explicitly
    setting the validate parameter to False.

The Dispatch.is_valid() method will return True or False indicating
validity of the _dispatch.

The Dispatch.validate() and Dispatch.json_dumps() methods will call the
jsonschema.Draft4Validator() method to validate the _dispatch, which
will raise an exception if the _dispatch isn't valid.

Dispatch instance _dispatch values can be populated when the instance is
initialized, and can be updated using methods with various rules for
preserving, modifying or replacing existing values.

Several initialization and update methods accept a skip_validation
parameter with a default value of True, and will validate the
resulting dispatch unless this parameter is explicity set to False. For
example, when constructing Dispatch instances there will be cases where
the instance is only partially complete and is not yet fully valid,
while in other cases a known valid Dispatch instance is being modified
and should remain valid.

Other update methods accept a validation parameter with a default
value of True, and will validate the resulting dispatch unless this
parameter is explicitly set to False. These are generally methods like
Dispatch.add_timestamps(), which modify subelements of the calling
Dispatch instance that have their own subschemas.

In all cases where validation checks fail, the calling Dispatch will
not be modified.

In general, any time values for the section of a _dispatch dict are
passed in a parameter, the parameter can hold either the section
structure directly or the path to a JSON file containing the section
structure. Thus, e.g., the _meta parameter to an updating method may
be either the actual _meta field value or a string specifying the path
to a JSON file holding the value.

Modification and update methods::

    Dispatch.merge(dispatch)
    Dispatch.merge(**dispatch_dict)

    Set values of undefined elements of the calling Dispatch instance to
    the corresponding values of the indicated dispatch_dict or Dispatch
    instance. Keys in the merge source that already exist in the calling
    Dispatch instance will be ignored; the calling Dispatch instance's
    values for those keys will not be changed.

::

    Dispatch.merge(dispatch, noclobber=False)
    Dispatch.merge(**dispatch_dict, noclobber=False)

    Set value of elements of the calling Dispatch instance with the
    corresponding elements of the indicated dispatch_dict or Dispatch
    instance. If a key exists in both the calling Dispatch instance and
    the merge source, the value in the calling instance will be
    overwritten.

Dispatch.add_timestamp(key, timestamp)

  Add timestamps to the calling Dispatch instance. Duplicates of
  existing timestamps will be ignored, in compliance with the schema.

Dispatch.set_source(source)

  Set the calling Dispatch origin source.

Dispatch.add_target(target)

  Add targets to the calling Dispatch instance. Duplicates of existing
  targets will be ignored, in compliance with the schema.

Dispatch.set_generator(type, generator)

  Set the calling Dispatch origin generator field.

Dispatch.set_commentary_fields(commentary)

  Set the calling Dispatch's commentary fields as indicated by the
  fields in the commentary parameter.

Dispatch.append_commentary(commentary)

  Append the indicated commentary text to the corresponding commentary
  fields. For string valued commentary, the text will be appended to
  any existing text. For list-of-strings valued commentary, the text
  will be appended as a list element.

'''

    '''
    Validate a possible dispatch against the dispatch schema.

    Parameters:
    candidate -- a possible dispatch dict.

    Returns:
    True if jsonschema.validate indicates candidate is a valid instance

    return jsonschema.Draft4Validator(
        schema,
        format_checker=jsonschema.FormatChecker(),
        resolver=Dispatch.schema_resolver()

        if os.path.isfile(os.path.join(schema_path, f))

    schema_names = [
        # Reduce by 1 to miss the period separator.
        f[:-len(schema_file_suffix) - 1] for f in files
        if f[-len(schema_file_suffix):] == schema_file_suffix

    schemas = {}
    for name in schema_names:

                    os.path.join(
                        schema_path,
                        '.'.join([name, schema_file_suffix])

    @classmethod
    def _load_or_copy(self, entity):
        '''
        Load content from a json file or do a deep copy of content
        depending on whether entity is the name of a json loadable
        file.
        '''
        try:
            with open(entity, 'r') as fp:
                return json.load(fp)

        # TypeError if entity isn't a string.
        # IOError if entity is a string that isn't a filename.
        # ValueError if entity is a filename but the file isn't JSON.
        except (TypeError, ValueError, IOError) as err:
            return copy.deepcopy(entity)

    def _items_exist(self, candidate, existing_dicts):
        '''
        Return True if all items in candidate are already present in one
        of the dicts in the existing_dicts list, False otherwise. This is
        used to check and skip duplicate entries when desired.
        '''
        duplicate = False
        for existing in existing_dicts:
            duplicate = True
            for (key, value) in candidate.items():
                if (key, value) not in existing.items():
                    duplicate = False
                    continue

        return duplicate

        '''
        Add a target entry.

        Before modifying the Dispatch instance, the new target is first
        checked for schema validity. If this fails, a ValidationError is
        raised and the Dispatch instance is not modified.

        Duplicates of existing target entries will be skipped. In this
        context, a new target is considered a duplicate of an existing
        target if all the items in the new target are items in the existing
        target.
        '''
        target_data = self._load_or_copy(target)
        # Always check to make sure the target is valid.
        validate(target_data, schema_name='network_node')

        # We didn't get an Exception in validate(), so it's OK to proceed.
        if 'origin' not in self._dispatch:
            self._dispatch['origin'] = {}

        if 'targets' not in self._dispatch['origin']:
            self._dispatch['origin']['targets'] = []

        # Check that the new target isn't a duplicate.
        if self._items_exist(target_data, self._dispatch['origin']['targets']):
            return

        self._dispatch['origin']['targets'].append(target_data)

    def _update_commentary_fields(self, commentary, append=False):
        '''
        Update commentary fields.

        This helper method is called by set_commentary_fields and
        append_to_commentary_fields using the append parameter to
        control the operation.

        Including a field that's not defined in the schema for the
        commentary section will result in a ValueError.

        Before modifying the Dispatch instance, the new commentary is first
        checked for schema validity. If this fails, a ValidationError is
        raised and the Dispatch instance is not modified.
        '''

        commentary_data = self._load_or_copy(commentary)

        # Extract a list of the legal fields per the schema.
        # commentary_schema = self.schema['definitions']['commentary_schema']
        commentary_schema = self.schemas['commentary']

        legal_commentary_fields = commentary_schema['properties'].keys()

        commentary_string_fields = [
            f for (f, s) in commentary_schema['properties'].items()
            if 'type' in s and s['type'] == 'string'
            ]
        commentary_list_fields = [
            f for f in legal_commentary_fields
            if f not in commentary_string_fields
            ]

        illegal_fields = []

        for field in commentary_data:

            # Every field needs to be in the schema.
            if field not in legal_commentary_fields:
                illegal_fields.append(field)

            # We allow passing in a single string for setting these
            # fields, and wrap it in an array for schema compliance.
            elif (
                    field in commentary_list_fields
                    and not isinstance(commentary_data[field], list)
                    ):
                commentary_data[field] = [commentary_data[field]]

        # If any of the fields we were passed aren't legal per the schema,
        # raise a ValueError now.
        if len(illegal_fields) > 0:
            raise ValueError(
                'Illegal commentary fields: %s' % (illegal_fields)
                )

        # Always check to make sure the commentary is valid.
        validate(commentary_data, schema_name='commentary')

        # We didn't get an Exception in validate(), so it's OK to proceed.
        if 'commentary' not in self._dispatch:
            self._dispatch['commentary'] = {}

        for field in commentary_data:

            if append:
                if field not in self._dispatch['commentary']:
                    self._dispatch['commentary'][field] = (
                        [] if field in commentary_list_fields else ''
                        )
                self._dispatch['commentary'][field] += commentary_data[field]

            else:
                self._dispatch['commentary'][field] = commentary_data[field]

[docs]    def merge(
            self,
            other_dispatch=None,
            noclobber=True,
            skip_validation=True,
            _meta={},
            origin={},
            commentary={}
            ):
        '''
        Copy another dispatch's information into this dispatch.

        The source for copying can be specified either by passing
        another Dispatch instance as the first parameter, or by
        passing named parameters for each section of a _dispatch. If
        both types of source are passed, the values from the Dispatch
        instance are first merged, followed by those of the individual
        sections.

        Parameters:
            other_dispatch -- A Dispatch instance to merge from.

            _meta -- Specific _meta section values to merge.
            origin -- Specific origin section values to merge.
            commentary -- Specific commentary section values to merge.

            noclobber -- If True, fields that exist in sections of the
                calling Dispatch instance won't be changed. If False,
                such fields will be set to the new values.
                Default: True

            skip_validation -- If True, the _dispatch resulting from
                this cloning will not be validated against the Dispatch
                schema. If False, it will be validated. If validation
                fails a ValidationError will be raised and the original
                _dispatch will be left unaltered.
                Default: True

        '''

        # - - - - - - - - - - - - - - - - - - - - - - - - - - - -
        def _copy_required_field_values(source, dest, noclobber):
            '''
            Copy values from the source dispatch dict structure to the dest
            dispatch dict structure, making sure to respect the noclobber
            parameter.

            This alters dest, and returns None.
            '''
            copy_keys = {
                field: [
                    # If noclobber is True, we omit keys that already
                    # occur in dest.
                    k for k in field_value
                    if (not noclobber)
                    or (field not in dest)
                    or (k not in dest[field])
                    ]
                for (field, field_value) in source.items()
                if field in self._core_dispatch_fields
                }
            for field in copy_keys:

                if field not in dest:
                    dest[field] = {}

                for key in copy_keys[field]:
                    # We already weeded out clobbering, if needed.
                    dest[field][key] = self._load_or_copy(source[field][key])
        # - - - - - - - - - - - - - - - - - - - - - - - - - - - -

        # First we create a separate dispatch to construct our updates, so
        # we don't alter this instance's information if validation fails.
        workset = copy.deepcopy(self._dispatch)

        # If other_dispatch is defined, we first update with its information.
        if other_dispatch is not None:
            _copy_required_field_values(
                other_dispatch._dispatch,
                workset,
                noclobber
                )

        # Then we update with the dispatch fields explicitly defined.
        _copy_required_field_values(
            {
                '_meta': self._load_or_copy(_meta),
                'origin': self._load_or_copy(origin),
                'commentary': self._load_or_copy(commentary)
                },
            workset,
            noclobber
            )

        # If validation isn't skipped, we check validity.
        # This raises an exception if validation fails.
        if not skip_validation:
            validate(workset)

        # If everything is OK, we copy our constructed dispatch.
        self._dispatch = copy.deepcopy(workset)
